# Plano da Sprint 4 - Vulkanos Academy

## 1. Resumo das Sprints Anteriores

### Sprint 1: Configuração e Autenticação Básica (Semanas 1-2)
*   **Foco:** Configuração do ambiente, estrutura do projeto e autenticação de usuários via e-mail/senha.
*   **Progresso Verificado:** Entidades de usuário com roles (`User.cs`), serviços de autenticação (`AuthService.cs`, `AuthController.cs`) e páginas de login/registro (`Login.razor`, `Register.razor`) estão presentes, indicando que a base de autenticação e gerenciamento de usuários está estabelecida.

### Sprint 2: Gestão de Cursos e Conteúdo (Semanas 3-4)
*   **Foco:** Criação e gestão de cursos, módulos e aulas, incluindo upload de mídias e player de vídeo.
*   **Progresso Verificado:** Entidades de curso, módulo, aula e material (`Course.cs`, `Module.cs`, `Lesson.cs`, `LessonMaterial.cs`), seus DTOs, controladores e serviços correspondentes estão implementados. Páginas como `CourseManagement.razor` e `LessonPlayer.razor` sugerem que a funcionalidade de gestão e consumo de conteúdo está em desenvolvimento.

### Sprint 3: Interatividade e Progresso do Aluno (Semanas 5-6)
*   **Foco:** Interatividade (comentários/fórum), dashboard do aluno com progresso e autenticação social.
*   **Progresso Verificado:** Entidade de comentário (`Comment.cs`), controlador e serviço de comentários (`CommentsController.cs`, `CommentService.cs`) estão presentes. A entidade `Enrollment.cs` inclui campos para `Progress` e `CompletionDate`, e a página `Dashboard.razor` provavelmente exibe o progresso do aluno. A autenticação social foi um objetivo, mas não foi explicitamente verificada nos arquivos.

## 2. Escopo da Sprint 4 (Semanas 7-8)

A Sprint 4 focará na conclusão das funcionalidades essenciais para o MVP, conforme o roadmap original, com ênfase em certificados, métricas e refinamento geral.

### Objetivos Principais:
1.  **Geração Automática de Certificados de Conclusão (PDF):** Permitir que alunos que completaram um curso gerem e baixem um certificado em formato PDF.
2.  **Dashboard de Métricas para Produtores/Administradores:** Fornecer uma visão geral das métricas chave da plataforma para Produtores e Administradores.
3.  **Refinamento e Testes:** Realizar testes de integração e aceitação, além de melhorias de usabilidade e correção de bugs.

### Funcionalidades a Serem Entregues:

#### Backend (VulkanosAcademy.Api)
*   **Serviço de Geração de Certificados:**
    *   Criar um novo serviço (`CertificateService.cs`) para orquestrar a geração de certificados.
    *   Implementar a lógica para gerar um PDF dinâmico com base nos dados do aluno e do curso (nome do aluno, nome do curso, data de conclusão, etc.).
    *   Integrar com uma biblioteca de geração de PDF (ex: `QuestPDF`, `iTextSharp` ou similar).
    *   Armazenar o PDF gerado (ex: em um diretório local ou serviço de armazenamento de objetos) e atualizar o `CertificateUrl` na entidade `Certificate`.
*   **Endpoint para Geração de Certificados:**
    *   Adicionar um endpoint no `CertificatesController.cs` (ou estender um existente) para disparar a geração e o download do certificado.
*   **Serviço de Métricas:**
    *   Criar um novo serviço (`MetricsService.cs`) ou estender o `EnrollmentService.cs` para calcular métricas globais e por curso (ex: total de alunos, cursos ativos, taxa de conclusão, receita).
    *   Expor endpoints na API para que o frontend possa consumir essas métricas.

#### Frontend (VulkanosAcademy.Web)
*   **Página de Certificados (Aluno):**
    *   Criar uma página ou componente (`Certificates.razor`) onde o aluno possa visualizar seus certificados e iniciar o download.
    *   Integrar com o endpoint de geração de certificados do backend.
*   **Dashboard de Produtor/Administrador:**
    *   Aprimorar a página `ProducerStats.razor` para exibir as métricas fornecidas pelo backend.
    *   Garantir que as métricas sejam atualizadas dinamicamente e apresentadas de forma clara.
*   **Refinamento da UI/UX:**
    *   Revisar as páginas existentes para melhorias de usabilidade e consistência visual.
    *   Correção de bugs identificados durante os testes.

class CertificateService:
    def __init__(self, storage_service):
        self.storage_service = storage_service

    def generate_certificate_pdf(self, enrollment_id, user_name, course_title, completion_date):
        # Lógica para gerar o PDF do certificado
        # Isso é um placeholder. Em um ambiente real, usaria uma biblioteca como QuestPDF ou iTextSharp.
        certificate_content = f"""
        CERTIFICADO DE CONCLUSÃO

        Este certificado atesta que {user_name} concluiu com sucesso o curso
        '{course_title}' em {completion_date.strftime('%d/%m/%Y')}.

        Vulkanos Academy
        """
        # Simula o salvamento do PDF e retorna uma URL
        certificate_url = self.storage_service.upload_pdf(f"certificate_{enrollment_id}.pdf", certificate_content)
        return certificate_url

class MetricsService:
    def __init__(self, enrollment_repository, course_repository):
        self.enrollment_repository = enrollment_repository
        self.course_repository = course_repository

    def get_global_metrics(self):
        total_enrollments = self.enrollment_repository.get_total_enrollments()
        completed_enrollments = self.enrollment_repository.get_completed_enrollments()
        active_courses = self.course_repository.get_active_courses_count()
        # Adicionar mais métricas conforme necessário
        return {
            "TotalEnrollments": total_enrollments,
            "CompletedEnrollments": completed_enrollments,
            "ActiveCourses": active_courses
        }

    def get_course_stats(self, course_id):
        # Este método já existe no EnrollmentService do frontend, mas a lógica de backend precisa ser implementada
        # para calcular as estatísticas reais.
        # Placeholder para a lógica de cálculo de estatísticas de um curso específico
        total_enrollments = self.enrollment_repository.get_course_total_enrollments(course_id)
        completed_enrollments = self.enrollment_repository.get_course_completed_enrollments(course_id)
        average_progress = self.enrollment_repository.get_course_average_progress(course_id)
        completion_rate = (completed_enrollments / total_enrollments * 100) if total_enrollments > 0 else 0
        active_students = self.enrollment_repository.get_course_active_students(course_id)

        return CourseStatsDto(
            TotalEnrollments=total_enrollments,
            CompletedEnrollments=completed_enrollments,
            AverageProgress=average_progress,
            CompletionRate=completion_rate,
            ActiveStudents=active_students
        )
